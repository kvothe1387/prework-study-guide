The Agile manifesto

  • We are uncovering better ways of developing software by doing it and helping others do it.Through this work we have come to value,

  • Individuals and interactions over processes and tools,
  • Working software over comprehensive documentation,
  • Customer collaboration over contract negotiation,
  • Responding to change over following a plan.

  • That is, while there is value in the items on the right, we value the items on the left more.

The 12 Principals

  • Our highest priority is to satisfy the customer through early and continuous delivery of valuable software.
  • Welcome changing requirements, even late in development. Agile processes harness change for the customer's competitive advantage.
  • Deliver working software frequently, from a couple of weeks to a couple of months, with a preference to the shorter timescale.
  • Business people and developers must work together daily throughout the project.
  • Build projects around motivated individuals. Give them the environment and support they need, and trust them to get the job done.
  • The most efficient and effective method of conveying information to and within a development team is face-to-face conversation.
  • Working software is the primary measure of progress.
  • Agile processes promote sustainable development. The sponsors, developers, and users should be able to maintain a constant pace indefinitely.
  • Continuous attention to technical excellence and good design enhances agility.
  • Simplicity--the art of maximizing the amount of work not done--is essential.
  • The best architectures, requirements, and designs emerge from self-organizing teams.
  • At regular intervals, the team reflects on how to become more effective, then tunes and adjusts its behavior accordingly.

HTML

  • URL- uniform resource locator.
  • HTML to give the content structure.
  • CSS to give the content style.
  • JavaScript to handle logic & allow users to interact with website.

  • Element is a type of content on a webpage i.e text, image, header, list elemtnts.
  • Element types that are used to divide & describe a webpage ie Meta-related, Content divison, Semantic.
  • Declarations are not case sensitve.
  • HTML elements are case sensitive.
  • HTML attribute defines additional info about an element, always defined in the beginning element.

  • The Head element contains information about the webpage.
  • The Body element represents the visible content shown to the user.
  • The Title element defines the title for a webpage.
  • The Style element defines the CSS styles associated with a webpage.
  • The Base element defines the base URL for a webpage.
  • The Link element connects an external resource to the HTML documen.t
  • The Meta element defines metadata ie character set, description, key words, author, & viewport.
  • The Script element embeds client side scripts in HTML document.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.

Git

  • git init :initialize an existing directory as a git repo
  • git clone :retrieve an entire repo from a hosted location via URL

  • git status :checks what branch we are currently on
  • git add [file] :add a file as it looks now to next commit
  • git reset [file] :unstage a file while retaining the changes in working directory
  • git diff :diff of what is changed but not staged
  • git diff --staged :diff of what is staged but not yet committed
  • git commit -m [messgae] :commit your staged content as a new commit snapshot

  • git branch :list your branches
  • git branch [branch name] :create a new branch at the current commit
  • git checkout -b [branch-name] :creates a new branch and switches to it
  • git merge [branch] :merge the specified branch's history into the current one
  • git log : show all commits in the current branch's history

JavaScript

  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • API- Browser app programming interfaces provide functionality
  • JavaScript is case sensitive
  • Each line of .js code ends with a ;

  • String- To signify that the value is a sring, enclose it in ''or""
  • Boolean- A true/false value, true & false don't need quote marks
  • Array- A structure that allows you to store mulitple values in a single reference
  • Object- Can be anything

  • Operators- A mathematical symbol that produces a result based on two values
  • + -Add two numbers or combine two strings together
  • = -Assignment
  • === - Strict equallity - Performs a test to see if two values are equal & of the same data type - Returns a boolean result
  • ! and !== -Not/does not equal

  • Conditionals - Code structures used to test if an expression returns true or not "if...else"
  • The expression inside the if() is the test
  • If comparison returns true, the 1st block of code runs
  • If comparison is n ot true, the 2nd block of code after the else statement runs instead

  • Functions- A way of packaging functionality that you wish to reuse
  • Possible to define a body of code as a function that executes when you call the function name in your code
  • Often take arguments: bits of data they need to do their job
  • Arguments go inside the () separated by commas if there is more than one argument

  • Events- Code structures that listen for activity in the browser and run code in response

  • JavaScript is a dynamic language with dynamic types
  • Variables are not directly associated with any particular value type
  • Any variable can be assigned and reassigned
  • Weakly typed lang- Allows implicit type conversion when operation involves mismatched types instead of throwing errors

  • Primitive Values- All types except 'object' define immutable values represented directly at the lowest level of the lang
  • All primitive types except "null" can be tested by the "typeof" operator
  • "typeof null" returns "object" so one has to use "===null" to test for "null"
  • When a property is accesed on a primititve value, .js auto wraps the value into the corresponding wrapper object & acceses the property on the object instead

  • Operational chaining(?)- Acceses an object property or calls a function
  • If the object accessed or function called using this operator is "undefined or null", the expression short circuits & evaluates to "undefined" instead of throwing an error
  • "undefined"- Indicates the absence of a value
  • "null"- Indicates the absence of an object
  • The language usually defaults to "undefined" when something is devoid of value
  • "null" is a keyword, but "undefined" is a normal identifier that happens to be a global property

  • Number type- A double precision 64bit binary format IEEE 754 value
  • Capable of storing positive floating point numbers
  • "NaN"- Not a number

  • BigInt type- A numeric primitiive in .js that can represent integers with arbitrary magnitude
  • Can safely store & operate on large integers beyond the safe integer limit for numbers
  • Created by appending "n" to the end of an integer or by calling the "BigInt()" function
  • Cannot represent fractional numbers

  • String type- Represents textual data and is encoded as a sequence of 16bit unsigned integer values representing UTF-16 code units
  • Each element in the string occupies a position in the string
  • The lenght of a string is the number of UTF-16 code units in it
  • .JS strings are immutable- Once a string is created it is not possible to modify it
  • String methods create new strings based on the content of the current string

  • Symbol type- A unique & immutable primitive value & may be used as the key of an object property
  • The purpose of symbols is to create unique property keys that are guaranteed not to clash with keys from other code

  • Objects- A value in memory which is possibly referenced by an identifier
  • In .js objects are the only mutable values
  • Functions are also objects with the additional capability of being callable

  • Properties- in .js objects can be seen as a collection of properties
  • Object properties are equivalent to key value pairs
  • Property keys are either strings or symbols
  • Property values can be values of any type including other objects
  • Two types of object properties- data property & accessor property

  • Data property- Associate a key with a value
  • "value"- The value retrieved by a get access of the property, can be any .js value
  • "writable"- A boolean value indicating if the property can be changed with an assignment
  • "enumerable"- A boolean value indicating if the property can be enumerated by a "for...in" loop
  • "configurable"- A boolean value indicating if the property can be deleted, can be changed to an accessor property & can have its attributes changed

  • Accessor property- Associates a key with one of two accessor functions "get" & "set" to retrieve or store a value
  • "get"- A function called with an empty argument list to retrieve the property value whenever a "get" access to the value is performed
  • "set"- A function called with an argument that contains the assigned value. Executed whenever a specified property is attempted to be changed
  • "enumerable"- A boolean value indicating if the property can be enumerated by a "for...in" loop
  • "configurable"- A boolean value indicating if the property can be deleted, can be changed to a data property, & can have its attributes changed

  • Arrays- Regular objects for which there is a particular relationship between integer keyed property & the length property
  • Typed arrays- Present an array like view of an underlying binary data buffer, umbrella term for a range of data structures
  • Often used in conjunction with "ArrayBuffer" & "DataView"

  • Keyed collections- Take object references as keys
  • "Set" & "WeakSet"- Represent a collection of unique values
  • "Map" & "WeakMap"- Represent a collection of key value associations
  • "WeakMap/Set"- Only allow garbage collectable values as keys, which are either objects or non-registered symbols
  • Keys may be collected even when they remain in the collection

  • JSON- .js object notation
  • Builds universal data structures that can be transferred between different environments & even across languages

  • Type coercion- Often use a value of one type where another type is expected, & the language will convert it to the right type
  • Primitive coercion- Used where a primitive value is expected, but theres no strong preference for what the actual type should be. Usually when a string, number, or BigInt are equally acceptable
  • Primitve conversion calls "valueof() before "tostring()
  • "const"- Allows us to create variable that can't be reassigned a value
  • Use an assignment operator (=) to assign the specific data we want stored in our variable

  • A "for" loop uses the predictable pattern of indices to perform a task on all items in an array by allowing a single code block to be executed over & over
  • 1st statement determines the starting point for our loop
  • 2nd statement is the condition
  • Final statement is what allows the array to iterate over each item
  • The code "x++" is a shortcut of writing "x+1"

  • Function- A set of instructions that tells the computer how to perfom a task
  • Does not auto execute when the .js file is run
  • Must call a function by name in order to execute it
  • A function name is followed by ()
  • The code to be executed is written inside {}
